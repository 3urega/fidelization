Creo que ahora mismo estás cayendo en un error muy habitual:

> Estás diseñando la pantalla como si fuera una app de autenticación.

Pero Fideli no es una app de autenticación.

Es una app de:

* descubrir recompensas
* conseguir premios
* volver a tu cafetería favorita

---

# Lo que NO haría

```text
Fideli

Tu fidelización, en un solo lugar

[ Registrarse ]
[ Registrar negocio ]

¿Ya tienes cuenta?
```

Esto parece:

* un banco
* un ERP
* Stripe
* Notion

No genera ninguna emoción.

# Mi recomendación para el MVP

Pantalla inicial:

```text
FIDELI

Tus cafés favoritos te recompensan

Acumula sellos, consigue premios y disfruta de ventajas exclusivas.

[ Empezar ]

────────────────

¿Tienes un negocio?

Crea tu programa de fidelización en menos de 5 minutos.

[ Registrar negocio ]
```

Porque el 90% de las visitas serán:

```text
cliente → recompensa
```

y no:

```text
negocio → administración
```

La home debe vender la ilusión de conseguir premios, no el proceso de registrarse. Esa pequeña diferencia suele mejorar muchísimo la conversión y además encaja mucho mejor con toda la identidad visual que estás construyendo para Fideli.
